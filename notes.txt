Let's create .env file  and add the following variables:
    - `ACCESS_TOKEN_SECRET`
    - `REFRESH_TOKEN_SECRET`

Always include the .env file in the gitignore file

In `authController.js`, once we found the match, we need to create access token. We call `sign` method on the `jsonwebtoken` package. The `sign` method takes three arguments: the payload, the secret key, and the options object. We will use the payload to store the user's name, the secret key to store the access token secret, and the options object to set the expiration time to 15 minutes.

Also we need to create refresh token. We'll follow the same steps as we did for the access token, but we'll set the expiration time to 1 day.

Then we define `otherUsers` array which will store the users that are not the current user. We also define `currentUser` object which will store the current user's information and the refresh token.

We can send the access token and refresh token to the client. We need to store the access token in memory and not in  localStorage. We can use the `res.cookie` method to send the refresh token to the client.
