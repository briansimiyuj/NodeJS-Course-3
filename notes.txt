We now create `authController.js` to handle the signin logic

Let's create a `handleSignIn` route handler in `authController.js`: then we destructure the `req` object to get the `username` and `password` fields. If the `username` and `password` fields are empty, we send back 400 status code with error message. If they are not empty, we check if the username exists in the database. If it doesn't exist, we send back 401 status code with error message. 
    - 401 status code means unauthorized

If the username exists, we compare the password with the hashed password in the database. If the passwords match, we send back 200 status code with success message. If they don't match, we send back 401 status code with error message.
