Let's create a `refreshTokenController` that will handle the refresh token logic. Then we create `handleRefreshToken` function which takes request and response as arguments. We retreive cookies from the request object, if there no cookies we return a 401 status code.

If there are cookies, we set `refreshToken` to the value of `cookies.JWT`. Then find a user in the database with the `refreshToken` that we just set. If there is no user, we return a 403 status code.

We need to verify JWT, this is a callback function that takes `refreshToken`, `payload` which is the refresh token from .env file and `err` and `decoded` as arguments. If there is an error or if the found user name is not equal to the `decoded` name, we return a 403 status code.

If everything is fine, we generate a new access token and return it in the response as JSON.

Then we create a `refreshRouter` which is a GET route that calls `handleRefreshToken` function. Finally we use it as a middleware in the `script` file. We place it above the `verifyJWT` middleware.

NB: Cookies are always sent via https, so in `authController` we need to set `secure: false` to allow cookies to be sent in development mode.
